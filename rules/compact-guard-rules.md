# Compact Guard — Hookify Rules

Rules for protecting the compact guard system integrity.

## Rule: protect-snapshots

**Action:** block
**Matcher:** Edit|Write
**Pattern:** `~/.claude/compact-guard/`

Do not edit or overwrite snapshot files. They are generated by hooks and should be read-only. If a snapshot needs to be regenerated, use the `/snapshot` skill or trigger the PreCompact hook.

## Rule: warn-context-heavy

**Action:** warn
**Matcher:** Read
**Pattern:** Files larger than 500 lines being read into context

When reading very large files (>500 lines), warn that this consumes significant context and may trigger compaction sooner. Suggest reading specific line ranges instead.

## Rule: protect-compact-guard-hooks

**Action:** block
**Matcher:** Edit|Write
**Pattern:** `~/.claude/hooks/compact-guard-*.sh`

Do not modify installed compact guard hooks directly. Changes should be made in the source project and then copied via the installer.

## Integration

These rules can be added to hookify via:

```bash
# If using hookify plugin
hookify add protect-snapshots block "compact-guard" "Do not edit snapshot files"
hookify add protect-compact-guard-hooks block "compact-guard-hooks" "Edit source, not installed hooks"
```

Or manually in your hookify configuration.
