# Context Security — Hookify Rules

Rules for the Context Guard security layer.

## Rule: protect-security-config

**Action:** warn
**Matcher:** Edit|Write
**Pattern:** `~/.claude/context-guard/security/`

Security configuration files should be modified through the `/cg-security-config` skill, not edited directly. Direct edits may break the config format.

## Rule: protect-security-log

**Action:** block
**Matcher:** Edit|Write
**Pattern:** `security.jsonl`

Do not edit or overwrite the security event log. It is an append-only audit trail generated by security hooks.

## Rule: warn-sensitive-file-read

**Action:** warn
**Matcher:** Read
**Pattern:** `.env`, `.pem`, `.key`, `.pfx`, `id_rsa`, `credentials`

Reading sensitive files loads their content into the LLM context window, which may cause credential leakage. Consider if this read is necessary.

## Integration

These rules can be added to hookify via:

```bash
hookify add protect-security-log block "security.jsonl" "Security log is append-only"
hookify add warn-sensitive-file-read warn ".env|.pem|.key" "Sensitive file content enters context"
```
